Hardwarové prostředky

Komunikace s hardware

I/O zařízení

  • Základní rozhraní mezi OS a hardware
  • Využívá ovladače v jádře OS
  • Zajišťuje vstup/výstup dat přes řadiče

Systém přerušení

  • Mechanismus pro získání pozornosti CPU
  • Pracuje asynchronně k hlavnímu zpracování
  • Spravován pomocí rutin obsluhy přerušení

Hardware registry

Izolované registry

  • Klasická metoda I/O komunikace
  • Používá IN/OUT instrukce CPU
  • Oddělený adresní prostor od paměti

Paměťově mapované registry

  • Registry se jeví jako paměťové adresy
  • Přístupné přes standardní paměťové instrukce
  • Rychlejší ale dražší implementace

Řadiče zařízení

Funkce ovladače

  • Implementuje obsluhu přerušení
  • Poskytuje API pro komunikaci s jádrem
  • Běží v privilegovaném režimu jádra

Typy řadičů

  • Vstupní řadiče pro sběr dat
  • Výstupní řadiče pro vysílání dat
  • Kombinované řadiče pro obousměrnou komunikaci

Privilegovaný režim

Úrovně ochrany

  • Ring 0 pro operace jádra
  • Ring 3 pro uživatelské aplikace
  • Řídí přístup ke kritickým instrukcím

Systémová volání

  • Rozhraní pro privilegované operace
  • Využívá softwarová přerušení
  • Ověřuje oprávnění operací

Časovací systém

Hardwarový časovač

  • Generuje periodická přerušení
  • Řídí plánování systému
  • Podporuje zpožděné vykonávání operací

Typy časovačů

  • PIT (Programovatelný intervalový časovač)
  • HPET (Vysoce přesný časovač událostí)
  • TSC (Čítač časových značek)
