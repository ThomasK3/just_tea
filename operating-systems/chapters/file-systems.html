Souborové systémy

Základní koncepty

Souborový systém

  • Organizovaná struktura pro ukládání dat
  • Spravuje metadata a samotná data
  • Implementován jako součást operačního systému

Metadata

  • Informace popisující organizaci dat
  • Obsahují oprávnění, časy, velikosti souborů
  • Kritická pro konzistenci souborového systému

Klíčové typy systémů

FAT

  • File Allocation Table od Microsoftu
  • Jednoduchý systém s omezenou velikostí
  • Používaný především pro přenosná média

NTFS

  • Pokročilý systém pro Windows NT
  • Podporuje žurnálování a šifrování dat
  • Maximální velikost oddílu 256TB

ext4

  • Standardní systém pro Linux
  • Využívá i-nodes pro metadata
  • Podporuje extenty pro efektivní alokaci

Pokročilé vlastnosti

Žurnálování

  • Zaznamenává změny před jejich provedením
  • Umožňuje obnovení po pádu systému
  • Zvyšuje spolehlivost na úkor výkonu

Fragmentace

  • Rozdělení souboru do různých částí disku
  • Snižuje rychlost čtení a zápisu
  • Řešeno defragmentací nebo prevencí

Extenty

  • Souvislé bloky alokovaného prostoru
  • Snižují režii správy metadat
  • Zefektivňují práci s velkými soubory
