Správa paměti v operačních systémech

Základní principy

Operační paměť

  • Vnitřní paměť počítače pro běžící programy
  • Přímo adresovatelná procesorem pomocí sběrnice
  • Využívá cache pro zrychlení přístupu

Adresní prostor

  • Logické uspořádání paměti pro procesy
  • Začíná od adresy 0 pro každý proces
  • Překládá se na fyzické adresy v RAM

Metody přidělování paměti

Přidělování veškeré paměti

  • Nejjednodušší způsob správy paměti procesům
  • Využíváno v jednouživatelských systémech
  • Omezuje multitasking na jeden proces

Přidělování pevných bloků

  • Rozdělení paměti na pevné oblasti
  • Umožňuje základní multitasking
  • Vede k vnitřní fragmentaci

Přidělování proměnných bloků

  • Dynamické přidělování dle požadavků procesu
  • Flexibilní využití paměti
  • Způsobuje vnější fragmentaci

Virtuální paměť

Stránkování

  • Rozdělení paměti na stejně velké stránky
  • Umožňuje nesouvislé umístění procesu
  • Využívá tabulku stránek pro překlad adres

Segmentace

  • Logické rozdělení programu na segmenty
  • Segmenty mají různou velikost
  • Využívá dvojici segment:offset

Problémy správy paměti

Memory leak

  • Únik paměti při chybné dealokaci
  • Vede k vyčerpání dostupné paměti
  • Řeší se pomocí garbage collectoru

Memory thrashing

  • Nadměrné stránkování do/z disku
  • Způsobuje výrazné zpomalení systému
  • Vzniká při nedostatku fyzické paměti
