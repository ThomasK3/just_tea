NAT, DHCP a DNS

NAT (Network Address Translation)

Koncepce NAT

  • Překlad síťových adres mezi sítěmi
  • Umožňuje sdílení veřejné IP adresy
  • Používán pro připojení lokálních sítí

Port forwarding

  • Přesměrování portů na vnitřní servery
  • Umožňuje přístup z internetu dovnitř
  • Používán pro hostování služeb

Druhy NAT

  • Static, Dynamic a PAT překlady
  • Různé způsoby mapování adres
  • Používán podle potřeb organizace

DHCP (Dynamic Host Configuration Protocol)

Komunikace DHCP

  • Automatická konfigurace síťových zařízení
  • Používá proces DORA (Discover-Offer-Request-Acknowledge)
  • Zajišťuje dynamické přidělování IP adres

DHCP Relay

  • Přeposílání DHCP požadavků mezi sítěmi
  • Umožňuje centralizovanou správu adres
  • Používán v rozsáhlejších sítích

DHCP zprávy

  • Obsahují konfigurační parametry sítě
  • Přenáší IP adresy a nastavení
  • Distribuuje síťovou konfiguraci klientům

DNS (Domain Name System)

Doménové jméno

  • Hierarchický systém pojmenování v internetu
  • Překládá názvy na IP adresy
  • Usnadňuje přístup k síťovým službám

Typy DNS serverů

  • Kořenové, TLD a autoritativní servery
  • Tvoří hierarchickou strukturu systému
  • Zajišťují distribuovanou správu domén

FQDN

  • Plně kvalifikované doménové jméno
  • Jednoznačně identifikuje síťový uzel
  • Používá se pro přesnou adresaci
