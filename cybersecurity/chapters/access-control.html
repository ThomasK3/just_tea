Systémy řízení přístupu

Základní modely přístupu

Role-based Access Control

  • Řízení přístupu na základě rolí
  • Přiřazení oprávnění skupinám uživatelů
  • Příklad: Administrátor, účetní, uživatel

Rule-based Access Control

  • Řízení přístupu podle pravidel
  • Dynamické vyhodnocování podmínek přístupu
  • Příklad: Časové omezení přístupu

Access Control Lists

  • Seznam oprávnění pro objekty
  • Definice povolení/zamítnutí pro subjekty
  • Příklad: Souborová oprávnění NTFS

Pokročilé modely přístupu

Mandatory Access Control

  • Centrálně řízená bezpečnostní politika
  • Vynucené bezpečnostní úrovně
  • Příklad: SELinux v Linux systémech

Discretionary Access Control

  • Vlastník určuje práva k objektům
  • Flexibilní delegace oprávnění
  • Příklad: Unix oprávnění k souborům

Attribute-based Access Control

  • Rozhodování podle atributů subjektu/objektu
  • Komplexní podmínky přístupu
  • Příklad: Geolokační omezení přístupu

Implementační aspekty

Autentizační mechanismy

  • Ověření identity subjektu
  • Vícefaktorová autentizace
  • Příklad: Heslo + biometrie

Autorizační proces

  • Vyhodnocení oprávnění k přístupu
  • Kontrola proti definovaným politikám
  • Příklad: Kontrola ACL při přístupu

Auditování přístupů

  • Záznam přístupových událostí
  • Detekce porušení oprávnění
  • Příklad: Logování přístupu k datům

Správa přístupových práv

Princip nejmenších privilegií

  • Minimální nutná oprávnění
  • Prevence zneužití práv
  • Příklad: Omezení admin přístupů

Separace povinností

  • Rozdělení kritických operací
  • Vyžadování více oprávnění
  • Příklad: Schvalování ve dvou osobách

Revize přístupů

  • Pravidelná kontrola oprávnění
  • Odebírání nepoužívaných práv
  • Příklad: Čtvrtletní audit přístupů
